# CrudApp

Painel de gestão de usuários construído com **React + Vite + Material UI + Supabase**.

CRUD completo (criar, listar, editar, ativar/desativar, excluir) com dashboard de estatísticas, máscara monetária BRL, validação de formulário, notificações toast contextuais e design responsivo.

## Stack

- **[React 18](https://react.dev/)** — biblioteca UI
- **[Vite 8](https://vitejs.dev/)** — build tool e dev server (com plugin-react 6 / OXC)
- **[Material UI 5](https://mui.com/)** — sistema de componentes e theming
- **[React Router 6](https://reactrouter.com/)** — roteamento
- **[Supabase](https://supabase.com/)** — banco de dados PostgreSQL + REST API + Row Level Security
- **[React Toastify 10](https://fkhadra.github.io/react-toastify/)** — notificações
- **[@fontsource/roboto](https://fontsource.org/fonts/roboto)** — fonte Roboto self-hosted

## Funcionalidades

- 📊 **Dashboard** com cards de estatísticas (total de usuários, folha de salários, ativos)
- 👥 **CRUD de usuários** — cadastrar, editar, excluir, ativar/desativar
- 💰 **Máscara monetária BRL** em tempo real no campo de salário
- 🎨 **Design responsivo** — sidebar adapta para drawer mobile com hamburguer
- 🌈 **Toasts coloridos** por tipo (success / warning / error) com transição slide
- 🔒 **Row Level Security** habilitado no Supabase
- ⚡ **Hot Module Replacement** via Vite + Rolldown

## Pré-requisitos

- **Node.js** 18 ou superior
- Uma conta gratuita no **[Supabase](https://supabase.com/)**

## Configuração

### 1. Clonar o repositório

```bash
git clone https://github.com/cielioqueiroz/crud-user-react.git
cd crud-user-react
```

### 2. Instalar dependências

```bash
npm install
```

### 3. Configurar o Supabase

#### 3.1 Criar projeto

1. Acesse o [dashboard do Supabase](https://supabase.com/dashboard) e crie um novo projeto
2. Escolha uma região (recomendado: **South America (São Paulo)** para projetos brasileiros)
3. Anote a senha do banco em local seguro

#### 3.2 Criar a tabela `usuarios`

No **SQL Editor** do projeto, execute:

```sql
CREATE TABLE usuarios (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  salario NUMERIC NOT NULL,
  funcao TEXT NOT NULL,
  email TEXT NOT NULL UNIQUE,
  contato TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'active',
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE usuarios ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Leitura pública" ON usuarios FOR SELECT USING (true);
CREATE POLICY "Insert público"  ON usuarios FOR INSERT WITH CHECK (true);
CREATE POLICY "Update público"  ON usuarios FOR UPDATE USING (true) WITH CHECK (true);
CREATE POLICY "Delete público"  ON usuarios FOR DELETE USING (true);
```

> ⚠️ As policies estão liberadas (`USING (true)`) porque esta aplicação demo não tem autenticação. Em produção, restrinja por `auth.uid()` ou outras condições.

#### 3.3 Configurar variáveis de ambiente

Copie o arquivo de exemplo:

```bash
cp .env.example .env
```

Preencha com os valores do seu projeto (em **Settings → API Keys**):

```
VITE_SUPABASE_URL="https://<seu-project-ref>.supabase.co"
VITE_SUPABASE_KEY="sb_publishable_..."
```

### 4. Rodar em desenvolvimento

```bash
npm run dev
```

A aplicação abre em [http://localhost:3000](http://localhost:3000).

## Decisões de design

- **Paleta**: slate `#0f172a` como primary, indigo `#4f46e5` como accent, com semânticas em emerald/amber/red
- **Tipografia**: Roboto com letter-spacing negativo nos títulos para feel premium
- **Sombras**: subtis em surfaces (Drawer, AppBar, Paper), mais profundas em dialogs
- **Mobile-first**: drawer temporário com hamburguer abaixo de `md`, grid responsivo nos stat cards
- **Toasts semânticos**: verde para sucesso, amarelo para warnings (validação, desativação), vermelho para erros e exclusão

## Segurança

- O `.env` está no `.gitignore` — nunca commite credenciais
- A `publishable_key` do Supabase é desenhada para uso no browser (segura desde que RLS esteja ativo)
- Nunca compartilhe a `secret_key` (`sb_secret_*`) do Supabase — ela é exclusiva para servidor
